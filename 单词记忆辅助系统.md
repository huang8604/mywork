# 单词记忆辅助系统总纲

## 1. 产品目标与边界

构建一个面向个人或家庭使用、部署在 NAS 上的轻量级单词记忆辅助系统，提供：

- 单词录入、检索、编辑、删除、CSV/JSON 导入与导出；
- 响应式 Web 界面，同一套页面支持手机、平板和电脑；
- 基于新词、错词、到期词和自定义词生成可打印的单词复习表；
- 以打印后的线下复习为主要使用方式，在线卡片复习作为偶尔使用的辅助方式；
- 将每次结果记录为“认识、不认识、跳过”，并允许之后手动修改；
- 复习流水、汇总统计、单词复习表和复习历史查询；
- 提供稳定、可发现、可授权的 REST API，允许外部 Skill 生成复习表和录入复习结果；
- GitHub Actions 构建镜像，NAS 运维人员通过 Portainer 手动更新。

系统默认是单用户、可信家庭网络内使用的 MVP。若经公网反向代理开放，必须启用 HTTPS、访问控制、受限 CORS 和备份策略。多用户、OCR 和高级自适应算法不在首版范围内。

## 2. 全局技术决策

| 领域 | 决策 |
| --- | --- |
| 后端 | Python、FastAPI、SQLAlchemy、Pydantic |
| 数据库 | SQLite，启用外键、WAL、busy timeout，并使用版本化迁移 |
| 前端 | Vue 3、TypeScript、Vite、Vue Router、Pinia |
| API | REST，统一前缀 `/api/v1`，OpenAPI 为唯一契约来源；网页和外部 Skill 复用相同领域接口 |
| 时间 | 数据库存储 UTC，界面按用户配置时区展示和计算“今日” |
| 复习结果 | `known`、`unknown`、`skipped`；允许手动修改 |
| 单词复习表 | 生成持久化复习表会话，保存参数、随机种子、题目快照和入选原因 |
| 部署 | 单容器运行；SQLite 数据目录挂载到 NAS |
| 发布 | CI 推送 GHCR `latest` 与不可变 SHA 标签；Portainer 人工 Pull latest + Recreate |

## 3. 核心业务规则

### 3.1 复习结果

- “认识”表示本次无需提示即可回忆成功。
- “不认识”表示回忆失败或需要查看答案。
- “跳过”表示本次不参与正确率和记忆间隔计算，但计入会话完成情况。
- 用户可在当前卡片、练习回录页或历史记录页修改结果。
- 修改既有结果时，系统必须在同一事务内更新流水和汇总统计，保留修改时间。
- 客户端为每次提交生成幂等键，网络重试不得重复累计。

### 3.2 选词策略

- 新词：不存在有效复习流水的单词。
- 错词：综合近期失败、连续失败和错误率排序，不仅按累计失败次数。
- 到期词：根据上次结果、连续成功和复习间隔计算 `due_at`，固定“若干天未复习”仅作为可配置兜底。
- 自定义词：`is_custom=true` 的单词，可设置独立配额。
- 同一个词只出现一次；若分类重叠，返回全部来源标签并按优先级补足配额。
- 响应包含请求数量、实际数量、随机种子和每个词的入选原因，结果可复现。

### 3.3 单词复习表闭环

1. 用户配置新词、错词、到期词和自定义词的数量与阈值。
2. 后端生成并保存一张单词复习表及其单词快照，前端提供打印预览。
3. 用户通常将复习表打印出来，在线下完成复习。
4. 每次线下复习后，用户在手机或电脑打开对应复习表，逐个记录本次结果：“认识 / 不认识 / 跳过”。
5. 每次实际复习都新增一组复习流水；如果只是纠正本次录入错误，则修改对应流水，不覆盖更早的复习历史。
6. 用户偶尔也可以使用在线卡片复习；线上和线下使用同一套三态结果、统计和修改规则。
7. 系统保存每次结果、刷新统计，并在历史页按复习表和时间展示。

### 3.4 外部 Skill 集成边界

- 外部 Skill 是受认证的 API 客户端，不模拟网页操作，也不能直接读写 SQLite。
- 系统公开 `/.well-known/word-review-api`、`/openapi.json` 和受认证的 `/api/v1/capabilities`，供 Skill 发现 API 版本、状态枚举、限制和 OpenAPI 地址。
- Skill 使用独立 Bearer Token，并只授予完成任务所需的作用域，例如 `practice:generate`、`practice:read`、`reviews:write`。
- 生成单词复习表必须调用与网页相同的生成服务，持久化 `practice_session`、题目快照、随机种子和来源。
- 录入结果必须先创建新的复习轮次，再逐项或批量提交 `known / unknown / skipped`；所有结果经 `review_service` 原子更新流水和统计。
- 生成会话和创建轮次使用 `Idempotency-Key`；每项结果使用 `client_event_id`。重试不得生成重复会话、轮次或统计。
- 每次 Skill 写操作记录 `api_client_id`、Skill 名称/版本、作用域、request_id 和业务对象 ID，日志不得记录完整 Token。
- API v1 只允许兼容性增加；破坏性变更发布到 `/api/v2`，并提供明确弃用期。

两个首要 Skill 场景：

1. **生成单词复习表 Skill**：读取能力描述，调用生成接口，取得 `session_id`、题目、打印页 URL 和数量摘要。
2. **录入复习结果 Skill**：为指定复习表创建轮次，批量录入三态结果；若只是纠正本轮错误，则修改对应流水。

## 5. 全局验收标准

- 320、375、768、1024、1440 像素宽度下核心流程可用；页面本身无横向溢出，宽表格仅在自身容器滚动。
- 所有关键操作可用键盘完成，焦点清晰，触控目标不小于 44×44 CSS 像素。
- 新增、修改、删除、导入、导出、复习提交、结果修改、练习生成、历史查询均通过 API 集成测试。
- `known / unknown / skipped` 三态统计正确，重复请求不重复计数，修改结果后可从流水重建一致统计。
- 两个示例 Skill 可只依赖 OpenAPI 完成“生成复习表”和“创建轮次并批量录入结果”；重复调用保持幂等，越权调用返回 403。
- 外部 Skill 的写操作可追溯到具体 API 客户端，不存在绕过 service 事务直接写数据库的入口。
- 打印在 Chrome/Edge 的 A4 纵向多页场景中不截行、重复表头且不打印导航控件。
- 空库和旧版本数据库均能通过迁移启动；备份恢复经过演练。
- CI 测试通过后才发布镜像；Portainer 人工更新后以健康检查确认成功，可用上一 SHA 镜像回滚。
