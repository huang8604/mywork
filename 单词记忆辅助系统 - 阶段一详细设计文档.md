# 单词记忆辅助系统－阶段一详细设计

> 状态：设计基线已冻结  
> 适用版本：API v1 / 数据库 schema v1  
> 最后更新：2026-07-20  
> 上游依据：[系统总纲](./单词记忆辅助系统.md)  
> 下游文档：阶段二后端、阶段三前端、阶段四打印、阶段五部署

## 1. 阶段目标、范围与出口

阶段一负责冻结实现边界，而不是编写业务代码。它向后续阶段交付：

1. 系统上下文、信任边界与组件职责；
2. 领域术语、不变量和状态机；
3. 可直接转成迁移脚本的实体、字段、约束、索引和删除语义；
4. REST API v1 的资源、参数、鉴权、幂等、并发、成功与错误语义；
5. 页面路由、数据依赖和响应式基线；
6. 非功能约束、验收矩阵以及 OpenAPI 落盘要求。

阶段一不包含 Python/Vue 工程、数据库迁移、容器和 CI 的实现。实现顺序与验收方式由后续阶段文档规定，但不得自行改变本文冻结的领域语义。

### 1.1 冻结决策

| 编号 | 决策 |
| --- | --- |
| D-01 | MVP 是单一共享词库，不建 user_id，不提供多用户隔离；未来多用户通过新 API 版本和迁移引入 |
| D-02 | 网页操作者由受信反向代理提供身份；应用不在 MVP 内实现账号、密码或登录会话 |
| D-03 | 外部 Skill 使用独立 Bearer Token 和最小 scope，不共享网页身份 |
| D-04 | review_logs 是复习事实源，word_stats 只是可重建缓存 |
| D-05 | 单词只软删除，normalized_en_word 在已删除和未删除记录之间仍保持全局唯一；再次使用必须恢复原记录 |
| D-06 | 同一张复习表可以多次使用；每次实际复习创建一个 review round，每个 round/item 最多一条 review_log |
| D-07 | known、unknown、skipped 是唯一结果枚举；skipped 计入完成度和跳过次数，不进入正确率或间隔计算 |
| D-08 | HTTP 请求幂等键与实际复习事件 ID 是两个概念；前者去除网络重试，后者标识一次真实作答 |
| D-09 | 纠错原地修改已有 review_log，使用 version 乐观锁，并从该单词全部流水重算统计 |
| D-10 | 数据库存储 UTC；MVP 只有一个 APP_TIMEZONE，默认 Asia/Shanghai，用于界面和“今日”边界 |
| D-11 | API v1 只允许兼容性增加；破坏性变更进入 v2 |
| D-12 | OpenAPI 3.1 文件是机器契约，本文是其人类可读设计；两者不一致时 CI 必须失败 |

### 1.2 不在阶段一和 MVP 范围内

- 多用户、家庭成员隔离和细粒度网页角色；
- OCR、语音评测、自动翻译和第三方词典抓取；
- SM-2、FSRS 等复杂自适应算法；
- 离线写入队列和冲突自动合并；
- 自动更新 NAS、自动回滚和 PDF 服务端渲染；
- 供 Skill 使用的任意 SQL、Shell、文件系统或任意 URL 请求能力。

## 2. 系统上下文与信任边界

### 2.1 组件

| 组件 | 职责 |
| --- | --- |
| Vue SPA | 词库管理、在线复习、复习表生成/打印/回录、历史与统计 |
| FastAPI | REST、认证授权、输入校验、事务、静态文件和 SPA 回退 |
| Domain services | normalization、review、stats、strategy、import/export、idempotency |
| SQLAlchemy repositories | 单表或明确查询的持久化；不拥有跨表业务事务 |
| SQLite | 唯一持久化数据库；启用 foreign_keys、WAL、busy_timeout |
| NAS 反向代理 | HTTPS、网页访问认证、可信身份头、请求体限制和公网入口控制 |
| 外部 Skill | 只通过 HTTPS REST API 调用被授权的业务能力 |

### 2.2 数据流

    浏览器
      │ HTTPS；由反向代理注入可信网页身份
      ▼
    NAS 反向代理 ─────────────┐
      │                       │
      │                       │ HTTPS + Bearer Token
      ▼                       ▼
    FastAPI / Vue 静态站点   外部 Skill
      │
      │ API → service → repository
      ▼
    SQLite 数据卷

信任边界规则：

- 反向代理必须删除客户端自带的身份头，只能由代理重新注入；应用仅信任 TRUSTED_PROXY_CIDRS 中的来源。
- 生产环境不得让公网绕过反向代理直连容器端口。
- 网页写请求必须校验 Host 和 Origin 与 PUBLIC_BASE_URL 一致。若反向代理使用 Cookie 登录，还必须使用 Secure、HttpOnly、SameSite，并提供其自身的 CSRF 防护。
- 外部 Skill 身份只来自 Bearer Token，不能由请求体自报客户端名称、版本或 scope。
- CORS 不是认证；服务端到服务端的 Skill 调用不依赖 CORS。

## 3. 领域术语与不变量

| 术语 | 定义 |
| --- | --- |
| 单词 word | 共享词库中的一个词条；全局 normalized_en_word 唯一 |
| 复习流水 review log | 一次实际作答的事实记录；纠错会修改它，不另建一条 |
| 复习统计 word stats | 从全部 review_logs 推导出的缓存 |
| 复习表 practice session | 一次选词生成结果，包含固定题目顺序和内容快照 |
| 题目 session item | 某个 session 中的一道题 |
| 复习轮次 review round | 使用同一张复习表进行的一次实际线下或线上复习 |
| 事件 ID client_event_id | 客户端为一次实际作答生成的稳定 ID |
| 幂等键 Idempotency-Key | 客户端为一次 HTTP 写请求生成的稳定 ID |
| 纠错 correction | 在同一 round/item 上修改已有结果；不代表新的复习 |

必须始终成立的不变量：

1. 同一 actor 的 client_event_id 只能对应一个业务事件。
2. 同一 review round 和 session item 最多有一条 review_log。
3. practice_session_items 的快照在创建后不可修改。
4. round、item、word 和 review_log 必须属于同一个 session/word 关系。
5. 任何复习写入或纠错都在同一事务中更新流水、题目最新结果、round 进度和 word_stats。
6. 软删除单词不能再产生新复习、进入新复习表或被普通查询导出，但历史和快照仍可查询。
7. 已归档 session 不能创建新 round；已完成 round 只能纠错，不能新增未曾作答的 item。
8. 所有列表都使用稳定的二级排序 id，避免分页时结果漂移。

## 4. 数据存储通用约定

### 4.1 SQLite 表示

- 主键：INTEGER，由 SQLite 生成。
- 布尔值：INTEGER NOT NULL，CHECK 值为 0 或 1。
- 时间：TEXT，格式为 RFC 3339 UTC，固定输出为 YYYY-MM-DDTHH:MM:SS.ffffffZ。
- JSON：TEXT，写入前按 UTF-8、对象键字典序、无无意义空白规范化；同时保存 schema/version 或请求摘要。
- 枚举：TEXT NOT NULL + CHECK，API 只接受文档列出的精确小写值。
- 所有 created_at、updated_at 由服务端时钟填写；updated_at 不早于 created_at。
- API 可接受带时区的 RFC 3339 输入，入库前转成 UTC；无时区时间返回 422。
- reviewed_at 最多允许比服务器当前时间晚 5 分钟，超出返回 422。

每个连接必须执行：

    PRAGMA foreign_keys = ON;
    PRAGMA journal_mode = WAL;
    PRAGMA busy_timeout = 5000;

数据库文件必须位于支持可靠 POSIX 文件锁的本地 NAS 文件系统，不得放在不可靠的 NFS/SMB 网络挂载上。迁移和 CI 必须执行 foreign_key_check 与 integrity_check。

### 4.2 文本规范化与长度

en_word 的保存规则：

1. Unicode NFKC；
2. 去除首尾空白；
3. 连续 Unicode 空白折叠为一个 ASCII 空格；
4. en_word 保存 1–200 个 Unicode code point 的展示值；
5. normalized_en_word 对展示值执行 Unicode casefold，长度仍不得超过 200；
6. 规范化为空串则拒绝。

tag 名称使用相同的 NFKC、trim、空白折叠和 casefold 规则，展示名长度为 1–50。中文释义最多 2000，音标最多 200，例句最多 5000，入选原因最多 500。所有文本均视为不可信纯文本。

### 4.3 ERD

    words 1 ─── 0..1 word_stats
      │ 1
      ├──── * word_tags * ─── 1 tags
      ├──── * review_logs
      └──── * practice_session_items * ─── 1 practice_sessions
                         │                         │
                         │                         └──── * practice_review_rounds
                         │                                      │
                         └──────── review_logs ──────────────────┘

    api_clients 1 ─── * api_client_scopes
         │
         ├──── * api_client_tokens
         └──── * audit_logs

    actors ─── * idempotency_records

actors 是逻辑上的多态身份，由 actor_type + actor_id 表示，不单独建表。

## 5. 数据模型

### 5.1 words

| 字段 | 类型与约束 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 主键 |
| en_word | TEXT NOT NULL | 规范空白后的展示拼写，长度 1–200 |
| normalized_en_word | TEXT NOT NULL UNIQUE | 全局规范键，软删除后仍占用 |
| phonetic | TEXT NULL | 长度 1–200；空串转 NULL |
| cn_meaning | TEXT NOT NULL | 长度 1–2000 |
| example_sentence | TEXT NULL | 长度 1–5000；空串转 NULL |
| is_custom | INTEGER NOT NULL DEFAULT 0 CHECK | 0 或 1 |
| version | INTEGER NOT NULL DEFAULT 1 CHECK | 大于 0；每次修改、删除或恢复加 1 |
| created_at | TEXT NOT NULL | UTC |
| updated_at | TEXT NOT NULL | UTC |
| deleted_at | TEXT NULL | 软删除时间 |

约束和行为：

- UNIQUE(normalized_en_word) 是全局唯一，不使用仅针对未删除记录的部分唯一索引。
- 新增命中已删除单词时返回 409 WORD_DELETED，并给出可恢复的 word_id；系统不创建第二个同名词。
- 普通列表、详情、生成、导入和导出默认排除 deleted_at 非空记录。
- 删除后历史详情仍通过 review/session 历史返回快照；普通 GET /words/{id} 返回 404。
- 不提供硬删除业务 API。

索引：

- UNIQUE(normalized_en_word)
- INDEX(deleted_at, created_at DESC, id DESC)
- INDEX(is_custom, deleted_at, id)

### 5.2 tags 与 word_tags

tags：

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| name | TEXT NOT NULL，长度 1–50 |
| normalized_name | TEXT NOT NULL UNIQUE |
| created_at | TEXT NOT NULL |

word_tags：

| 字段 | 类型与约束 |
| --- | --- |
| word_id | INTEGER NOT NULL FK → words.id ON DELETE CASCADE |
| tag_id | INTEGER NOT NULL FK → tags.id ON DELETE CASCADE |

PRIMARY KEY(word_id, tag_id)，另建 INDEX(tag_id, word_id)。标签不是独立业务资源；新增/更新单词时按规范键复用，未使用标签可由维护命令清理。

### 5.3 review_logs

| 字段 | 类型与约束 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 流水 ID |
| word_id | INTEGER NOT NULL FK → words.id ON DELETE RESTRICT | 关联单词 |
| session_item_id | INTEGER NULL FK → practice_session_items.id ON DELETE RESTRICT | 复习表题目 |
| review_round_id | INTEGER NULL FK → practice_review_rounds.id ON DELETE RESTRICT | 本次轮次 |
| status | TEXT NOT NULL CHECK | known / unknown / skipped |
| source | TEXT NOT NULL CHECK | quick_review / online_practice / print_manual |
| actor_type | TEXT NOT NULL CHECK | web_user / api_client |
| actor_id | TEXT NOT NULL | 1–128 字符 |
| client_event_id | TEXT NOT NULL | 1–128 字符，推荐 UUID |
| duration_ms | INTEGER NULL CHECK | 0–86400000 |
| reviewed_at | TEXT NOT NULL | 实际作答时间，UTC |
| version | INTEGER NOT NULL DEFAULT 1 CHECK | 大于 0，纠错后加 1 |
| created_at | TEXT NOT NULL | 首次写入时间 |
| updated_at | TEXT NOT NULL | 最后纠错时间 |

数据库约束：

- UNIQUE(actor_type, actor_id, client_event_id)
- session_item_id 与 review_round_id 必须同时为空或同时非空。
- source=quick_review 时两个关联字段必须为空。
- source=online_practice 或 print_manual 时两个关联字段必须非空。
- 对非空关联建立 UNIQUE(review_round_id, session_item_id)。

跨表不变量由 review_service 在事务内校验：

- round.session_id = item.session_id；
- review_log.word_id = item.word_id；
- offline round 只能产生 print_manual，online round 只能产生 online_practice；
- 软删除 word 不允许新增流水；
- api_client 只能纠正 actor_type/api_client 且 actor_id 为自身 ID 的流水；web_user 可纠正全部流水。

索引：

- INDEX(word_id, reviewed_at DESC, id DESC)
- INDEX(status, reviewed_at DESC, id DESC)
- INDEX(source, reviewed_at DESC, id DESC)
- INDEX(review_round_id, session_item_id)
- INDEX(actor_type, actor_id, reviewed_at DESC)

### 5.4 word_stats

| 字段 | 类型与约束 |
| --- | --- |
| word_id | INTEGER PK FK → words.id ON DELETE CASCADE |
| known_count | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| unknown_count | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| skipped_count | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| consecutive_known | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| consecutive_unknown | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| last_status | TEXT NULL CHECK known/unknown/skipped |
| last_reviewed_at | TEXT NULL |
| last_effective_status | TEXT NULL CHECK known/unknown |
| last_effective_reviewed_at | TEXT NULL |
| interval_days | INTEGER NOT NULL DEFAULT 0 CHECK >= 0 |
| due_at | TEXT NULL |
| updated_at | TEXT NOT NULL |

附加不变量：

- consecutive_known 与 consecutive_unknown 至多一个大于 0。
- total_attempts = known_count + unknown_count + skipped_count，仅作为响应派生字段。
- accuracy 分母为 known_count + unknown_count；分母为 0 时返回 null。
- last_status/last_reviewed_at 包含 skipped；last_effective_* 忽略 skipped。
- skipped 不改变两个 consecutive 字段、interval_days、last_effective_* 和 due_at。
- 索引 INDEX(due_at, word_id) WHERE due_at IS NOT NULL。

统计按 reviewed_at ASC, id ASC 重放。修改 reviewed_at 或 status 后也使用该顺序完整重算该 word。允许修改 reviewed_at 是为了纠正延迟回录或时钟录入错误，可能改变该词后续连续次数、间隔与 due_at；界面必须明确提示这一影响，审计 metadata 记录修改前后的 status/reviewed_at/version。若统计更新失败，整个复习事务回滚。维护命令可在独占写窗口中重建全部统计，并逐字段比较后原子替换。

### 5.5 practice_sessions

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| status | TEXT NOT NULL DEFAULT active CHECK active/archived |
| strategy_version | TEXT NOT NULL | v1 |
| strategy_params_json | TEXT NOT NULL | 规范 JSON |
| strategy_hash | TEXT NOT NULL | SHA-256 十六进制 |
| seed | INTEGER NOT NULL CHECK 0–2147483647 |
| requested_counts_json | TEXT NOT NULL |
| actual_counts_json | TEXT NOT NULL |
| created_by_actor_type | TEXT NOT NULL CHECK web_user/api_client |
| created_by_actor_id | TEXT NOT NULL |
| skill_name | TEXT NULL |
| skill_version | TEXT NULL |
| version | INTEGER NOT NULL DEFAULT 1 CHECK > 0 |
| generated_at | TEXT NOT NULL |
| printed_at | TEXT NULL |
| completed_at | TEXT NULL | 第一轮完成时间；不阻止以后新建轮次 |

status 只表示资源是否还能创建新轮次。打印和复习完成不是互斥状态：printed_at 记录第一次确认打印的时间，completed_at 记录第一轮完成的时间。

索引：

- INDEX(status, generated_at DESC, id DESC)
- INDEX(created_by_actor_type, created_by_actor_id, generated_at DESC)

### 5.6 practice_session_items

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| session_id | INTEGER NOT NULL FK → practice_sessions.id ON DELETE CASCADE |
| word_id | INTEGER NOT NULL FK → words.id ON DELETE RESTRICT |
| position | INTEGER NOT NULL CHECK > 0 |
| snapshot_en_word | TEXT NOT NULL |
| snapshot_phonetic | TEXT NULL |
| snapshot_cn_meaning | TEXT NOT NULL |
| snapshot_example_sentence | TEXT NULL |

| reason | TEXT NOT NULL |
| latest_review_log_id | INTEGER NULL FK → review_logs.id ON DELETE SET NULL |

约束：

- UNIQUE(session_id, word_id)
- UNIQUE(session_id, position)
- 快照字段、position、来源和 reason 创建后不可修改。
- latest_review_log_id 只允许指向本 item 的、reviewed_at/id 最新的流水。

### 5.7 practice_review_rounds

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| session_id | INTEGER NOT NULL FK → practice_sessions.id ON DELETE CASCADE |
| mode | TEXT NOT NULL CHECK offline/online |
| status | TEXT NOT NULL DEFAULT open CHECK open/completed |
| created_by_actor_type | TEXT NOT NULL CHECK web_user/api_client |
| created_by_actor_id | TEXT NOT NULL |
| started_at | TEXT NOT NULL |
| completed_at | TEXT NULL |
| version | INTEGER NOT NULL DEFAULT 1 CHECK > 0 |
| created_at | TEXT NOT NULL |
| updated_at | TEXT NOT NULL |

索引：

- INDEX(session_id, started_at DESC, id DESC)
- INDEX(status, started_at DESC)

round 在所有 session item 都已有结果时自动转为 completed，completed_at 只写一次。completed round 允许纠正已有结果；不能为此前未作答的 item 新增结果。若用户只完成部分题目并结束，保持 open，稍后可继续。

### 5.8 api_clients、scopes 与 tokens

api_clients：

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| name | TEXT NOT NULL，1–100 |
| description | TEXT NULL，最多 500 |
| skill_name | TEXT NOT NULL，1–100 |
| skill_version | TEXT NOT NULL，1–50 |
| status | TEXT NOT NULL DEFAULT active CHECK active/disabled |
| scope_version | INTEGER NOT NULL DEFAULT 1 CHECK > 0 |
| created_at | TEXT NOT NULL |
| updated_at | TEXT NOT NULL |

api_client_scopes：

| 字段 | 类型与约束 |
| --- | --- |
| api_client_id | INTEGER FK → api_clients.id ON DELETE CASCADE |
| scope | TEXT NOT NULL CHECK 在固定白名单内 |

PRIMARY KEY(api_client_id, scope)。scope 变更原子提交、scope_version 加 1，并立即影响该客户端的所有 Token。

api_client_tokens：

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| api_client_id | INTEGER NOT NULL FK → api_clients.id ON DELETE CASCADE |
| token_prefix | TEXT NOT NULL | 仅用于人工识别，不是秘密 |
| token_hash | TEXT NOT NULL UNIQUE | Argon2id + 服务端 pepper |
| created_at | TEXT NOT NULL |
| expires_at | TEXT NOT NULL CHECK 晚于 created_at |
| last_used_at | TEXT NULL |
| revoked_at | TEXT NULL CHECK 不早于 created_at |

Token 至少 256 位随机熵，只显示一次。Argon2id 参数作为部署配置冻结并随哈希保存；参数升级时成功认证后可重哈希。认证必须恒时比较，并同时检查客户端状态、到期和撤销时间。完整 Authorization 永不进入应用或反向代理日志。

### 5.9 idempotency_records

| 字段 | 类型与约束 |
| --- | --- |
| id | INTEGER PK |
| actor_type | TEXT NOT NULL CHECK web_user/api_client |
| actor_id | TEXT NOT NULL |
| method | TEXT NOT NULL |
| route_template | TEXT NOT NULL | 使用模板，不使用带查询串的原始 URL |
| idempotency_key | TEXT NOT NULL | 1–128 字符 |
| request_hash | TEXT NOT NULL | 规范请求的 SHA-256 |
| state | TEXT NOT NULL CHECK processing/succeeded |
| response_status | INTEGER NULL |
| response_json | TEXT NULL | 仅保存小型 JSON envelope |
| resource_type | TEXT NULL |
| resource_id | TEXT NULL |
| created_at | TEXT NOT NULL |
| expires_at | TEXT NOT NULL |

UNIQUE(actor_type, actor_id, method, route_template, idempotency_key)，并建立 INDEX(expires_at)。

请求摘要覆盖 method、route template、规范 query、影响语义的 headers 和规范 JSON/body hash。相同键和相同摘要返回原结果；相同键但不同摘要返回 409 IDEMPOTENCY_KEY_REUSE。

幂等 claim、业务写入、response_json/resource 引用和 state=succeeded 必须处于同一个 SQLite 事务；processing 不能作为独立事务提交。因此进程在提交前崩溃会整体回滚，提交后但返回响应前崩溃则可由 succeeded 记录重放，不会留下永久 processing。并发请求先按有限退避等待首次事务；超时返回 409 REQUEST_IN_PROGRESS 和 Retry-After。未校验通过的 4xx 不占用键；5xx 回滚整个事务，以便安全重试。记录至少保留 30 天，配置不得低于 30。

## 6. 状态机、事务与算法

### 6.1 单词

    active ── DELETE ──> deleted
      ▲                    │
      └──── restore ───────┘

DELETE 和 restore 都要求 version。重复 DELETE 返回 204，不再次增加 version；恢复 active 单词返回 200 原资源。

### 6.2 复习表与轮次

    practice session: active ── archive ──> archived

    review round: open ── 全部题目有结果 ──> completed
                       completed ── correction ──> completed

- archived session 可读、可打印历史，但不能创建 round。
- session.completed_at 是第一轮完成时间，不是 session 终态。
- round 的 mode 创建后不可修改。
- completed round 只允许修改已有 review_log。

### 6.3 一次复习写事务

首次结果：

1. 完成身份、scope、session/round/item/word、状态和时间校验；
2. 检查 round 是否允许新增；
3. 尝试插入 review_log，依靠两个唯一约束处理并发；
4. 从该 word 的全部流水重算并 UPSERT word_stats；
5. 更新 item.latest_review_log_id；
6. 重新计算 round 进度，必要时完成 round 和首次完成 session；
7. 写审计和幂等结果；
8. 提交。

纠错结果：

1. 验证创建者权限和 client_event_id；
2. 以 UPDATE ... WHERE id = ? AND version = expected_version 原子更新；
3. 影响行数为 0 时返回 409 VERSION_CONFLICT，并附当前资源；
4. 从流水完整重算该 word_stats，再更新 item/round；
5. 写审计并提交。

任何步骤失败全部回滚。只对 SQLite BUSY/LOCKED 做最多 3 次、带抖动的 50/100/200ms 退避重试；仍失败返回 503 SERVICE_BUSY 和 Retry-After: 1。事务体必须可重放，不能在重试期间发送外部消息。

### 6.4 间隔与统计

按忽略 skipped 后的有效结果序列计算：

| 新结果 | consecutive_known | consecutive_unknown | interval_days |
| --- | ---: | ---: | ---: |
| unknown | 0 | 上次值 + 1 | 1 |
| known，第 1 次连续 | 1 | 0 | 1 |
| known，第 2 次连续 | 2 | 0 | 3 |
| known，第 3 次连续 | 3 | 0 | 7 |
| known，第 4 次连续 | 4 | 0 | 14 |
| known，第 5 次及以后 | 上次值 + 1 | 0 | 30 |
| skipped | 不变 | 不变 | 不变 |

known/unknown 的 due_at = reviewed_at + interval_days。只有 skipped 且此前无有效结果时，due_at 保持 null。统计摘要中的“今日”使用 APP_TIMEZONE 把 UTC 边界转换后查询。

### 6.5 选词策略 v1

请求字段：

| 字段 | 默认 | 范围 |
| --- | ---: | ---: |
| new_words_limit | 10 | 0–100 |
| error_words_limit | 15 | 0–100 |
| due_words_limit | 20 | 0–100 |
| custom_words_limit | 5 | 0–100 |
| fallback_unreviewed_days | 3 | 1–365 |
| seed | 服务端生成 | 0–2147483647 |

四类 limit 总和不得超过服务器 MAX_PRACTICE_WORDS，MVP 默认 200。

候选：

- new：不存在 review_log 的 active word；
- error：unknown_count > 0；
- due：due_at <= now；只有旧数据 due_at 为空时才使用 fallback_unreviewed_days；
- custom：active 且 is_custom=1。

其中“旧数据 due_at 为空”的精确定义是：至少有一条 review_log、last_effective_reviewed_at 非空、due_at 为空，且 last_effective_reviewed_at <= now - fallback_unreviewed_days。新词和只有 skipped 的词不进入该兜底候选。

选择优先级为 error → due → custom → new。同词重叠时只入选一次，source_categories 保留全部命中分类；被重叠占用的配额继续从该分类候选池补位。候选基础排序必须稳定，随机只使用 seed 对本次局部生成器生效。服务端在同一事务保存 session、参数、hash、seed、请求/实际数量、题目快照和原因。

错词评分从高到低：

    recent_unknown_7d * 100
    + consecutive_unknown * 20
    + floor(unknown_count * 10 / (known_count + unknown_count))
    + recency_bonus

recency_bonus 为最近一次 unknown 在 1/3/7/30 天内分别 8/6/4/2，否则 0；分段边界按小于等于计算。所有整数运算按上述 floor 规则执行，同分按 last_effective_reviewed_at ASC、word_id ASC。算法改变必须增加 strategy_version，历史 session 始终保留生成时版本和快照。

## 7. REST API v1 契约

### 7.1 路径、媒体类型与身份

- 业务前缀：/api/v1。
- JSON：application/json; charset=utf-8。
- 文件上传：multipart/form-data。
- 时间：RFC 3339，响应统一 UTC。
- request_id：服务端生成 UUID；只透传符合 UUID 格式的 X-Request-ID。
- 所有业务数据端点需要网页身份或 Bearer Token。
- /.well-known/word-review-api、/openapi.json、/healthz/live 公开且不得含敏感信息。
- /healthz/ready 不需要 Bearer，但只应由反向代理/内网探测，响应只给 ready 或 unavailable。

网页身份：

- 请求必须来自 TRUSTED_PROXY_CIDRS；
- 代理注入 X-Forwarded-User，应用映射为 actor_type=web_user；
- 受限本地部署可使用固定 actor_id=local-admin，但容器端口必须只对可信局域网开放；
- web_user 视为共享词库管理员，可调用全部业务端点。

Skill 身份：

- Authorization: Bearer token；
- actor_type=api_client，actor_id=api_clients.id；
- Token 无效、到期、撤销或客户端 disabled 返回 401 AUTH_REQUIRED；
- Token 有效但 scope 不足返回 403 FORBIDDEN_SCOPE。

### 7.2 Scope

| Scope | 权限 |
| --- | --- |
| words:read | 查询词条；查询词条统计时还需 reviews:read |
| words:write | 新增、更新、删除、恢复和导入 |

| practice:generate | 生成复习表 |

### 7.3 成功、分页和错误

JSON 成功 envelope：

    {
      "code": "OK",
      "message": "success",
      "data": {},
      "meta": {},
      "request_id": "UUID"
    }

列表 meta：

    {
      "page": 1,
      "size": 20,
      "total": 125
    }

page 默认 1、最小 1；size 默认 20、范围 1–100。所有列表排序最后追加 id；未知 query 参数返回 422。

错误 envelope：

    {
      "code": "VALIDATION_ERROR",
      "message": "请求参数不合法",
      "details": [
        {
          "path": ["body", "items", 1, "status"],
          "reason": "unsupported value",
          "value": "maybe"
        }
      ],
      "request_id": "UUID"
    }

生产环境 details 不回显 Token、路径、SQL、堆栈或完整文件内容。固定错误码：

| HTTP | code | 使用场景 |
| ---: | --- | --- |
| 400 | BAD_REQUEST | 语法有效但请求组合不成立 |
| 401 | AUTH_REQUIRED | 身份缺失或无效 |
| 403 | FORBIDDEN_SCOPE | scope 不足或无权纠错他人流水 |
| 404 | NOT_FOUND | 资源不存在、已软删除或不可见 |
| 409 | DUPLICATE_WORD | active 单词规范键重复 |
| 409 | WORD_DELETED | 同规范键单词已软删除 |
| 409 | IDEMPOTENCY_KEY_REUSE | 同键对应不同请求 |
| 409 | REQUEST_IN_PROGRESS | 同键请求尚未完成 |
| 409 | VERSION_CONFLICT | expected_version 过期 |
| 409 | INVALID_STATE | session/round 状态不允许操作 |
| 413 | PAYLOAD_TOO_LARGE | 文件、行数或请求体超限 |
| 415 | UNSUPPORTED_MEDIA_TYPE | Content-Type 或文件类型不支持 |
| 422 | VALIDATION_ERROR | 字段、枚举、边界或跨字段校验失败 |
| 429 | RATE_LIMITED | 超过客户端限流 |
| 500 | INTERNAL_ERROR | 未预期错误 |
| 503 | SERVICE_BUSY | SQLite 锁竞争超出重试 |

429 和 503 必须带 Retry-After。404 不区分“不存在”和“无读取权限”，避免枚举资源。

### 7.4 并发与幂等头

- 外部 API 客户端的所有 POST、PUT、PATCH 写请求都必须带 Idempotency-Key。
- 网页生成 session、创建 round、批量结果也必须带 Idempotency-Key；普通单项结果可依靠 client_event_id，但仍建议携带。
- 相同幂等请求重放返回与首次相同的 HTTP 状态和 envelope，并加 Idempotency-Replayed: true。
- client_event_id 首次创建后不可更换；纠错必须复用它。
- 需要纠错的请求必须带 expected_version。初始 version=1，每次成功纠错加 1。
- VERSION_CONFLICT 的 details 包含 current_version 和当前资源最小快照。

### 7.8 单词导入导出

导入 multipart 字段：

- file：CSV 或 JSON，UTF-8，可带 BOM；
- conflict_policy：skip / update / reject；
- dry_run：boolean，默认 false。

默认上限：10 MiB、10000 行；实际值由 capabilities 返回。压缩文件不接受。CSV 首行必须使用固定列 en_word、phonetic、cn_meaning、example_sentence、is_custom、tags，tags 使用分号分隔并转义；JSON 是 WordWrite 数组但不含 expected_version。

MVP 导入为原子模式：

1. 完整读取并校验大小、编码、表头、行数、所有字段与文件内重复；
2. 任一格式/字段错误返回 422，details 带 row/path，数据库零变更；
3. reject 遇到数据库重复返回 409，数据库零变更；
4. skip 跳过重复，update 更新重复 active word；update 是明确的整行覆盖语义，覆盖 phonetic、cn_meaning、example_sentence、is_custom、tags 并令 word.version+1，不改变 id/created_at；命中 deleted word 一律返回 409 WORD_DELETED，不自动恢复；
5. dry_run 只返回预测计数，不写入；
6. 非 dry_run 的冲突读取与写入使用同一个 BEGIN IMMEDIATE 事务，使并发导入串行；后进入的 update 按提交顺序覆盖前一次值，该行为写入审计；
7. 成功响应返回 created、updated、skipped、rejected、total，其中原子成功时 rejected 固定为 0；失败行放在错误 envelope 的 details 中。

导出：

- GET /words/export 接受 words 列表的相同过滤参数和 format=csv|json；
- 默认排除软删除；
- CSV 使用 UTF-8 BOM，固定列顺序；JSON 返回数组文件；
- 单元格首个非空字符为 =、+、-、@ 时前置单引号；
- 响应设置 Content-Disposition、X-Content-Type-Options: nosniff，不套 envelope。

actual_counts 的各分类表示入选词命中该分类的数量，因此可重叠；unique_total 才是题目数。没有候选时仍创建一个 items=[] 的可追溯 session 并返回 201，由页面显示 empty 状态且不渲染空表；空 session 不允许创建 review round，尝试时返回 409 INVALID_STATE。

GET /practice-sessions/{id} 返回快照、策略、printed_at、首次 completed_at、所有 round 摘要以及指定 active_round_id 的当前结果。打印 URL 不含 Token；打开页面仍需网页访问认证。

printed 接口无请求体，第一次调用写 printed_at，重复调用保持原时间并返回 200。archive 第一次归档，重复调用返回相同资源；归档不影响历史读取。

### 7.10 创建轮次与回录

创建请求：

    {
      "mode": "offline",
      "started_at": "2026-07-20T09:00:00Z"
    }

mode 必须为 offline/online；started_at 可省略并默认服务器时间。成功返回 round_id、session_id、status=open、version=1、item_total、answered_count=0。

单项 PUT 只使用路径中的 item_id，正式 OpenAPI 的单项 body 不包含 item_id；首次创建返回 201，纠错返回 200。批量 body 才包含每项 item_id。

批量请求：

    {
      "items": [
        {
          "item_id": 1001,
          "status": "known",
          "client_event_id": "evt-001"
        },
        {
          "item_id": 1002,
          "status": "unknown",
          "client_event_id": "evt-002"
        },
        {
          "item_id": 1003,
          "status": "skipped",
          "client_event_id": "evt-003"
        }
      ]
    }

批量上限默认 200，最少 1。MVP 只有原子模式：

- item_id 或 client_event_id 在同一请求内重复，返回 422；
- 所有 item、权限、版本和跨表关系在写入前校验；
- 任一项非法、冲突或越权，全部回滚，流水、统计、round 进度均不改变；
- 首次与纠错可以混合，但纠错项必须带 expected_version；
- 同一个 actor/client_event_id 的同内容重试返回原流水，不同内容返回 409；
- 响应包含 round 进度、每项 review_log_id/status/version/updated_at 和最新统计摘要。

### 7.12 OpenAPI 落盘规范

实现阶段必须提交 contracts/openapi.yaml，并由 FastAPI 构建时校验其等价输出：

- OpenAPI 3.1；
- operationId 稳定且唯一；
- securitySchemes 同时描述 Bearer 与名为 TrustedProxyUser、类型 apiKey、位置 header、名称 X-Forwarded-User 的网页身份；每个业务操作以 OR 关系允许其中一种身份。说明必须强调 TrustedProxyUser 只有在来源属于 TRUSTED_PROXY_CIDRS 且代理已剥离客户端同名头时才有效；
- 每个操作列出 scope、headers、参数、requestBody、成功响应和全部错误引用；
- 所有 schema 默认 additionalProperties=false；
- known/unknown/skipped、source、mode、status、sort、conflict_policy 全部使用 enum；
- Idempotency-Key、Idempotency-Replayed、If-Match/expected_version、Retry-After 明确到操作；
- JSON envelope、分页 meta、错误 details、文件响应和示例均可校验；
- CI 执行 lint、breaking-change diff、后端路由一致性和前端类型生成。

## 8. 页面信息架构与响应式基线

### 8.1 路由和数据依赖

| 路由 | 页面 | 主要 API |
| --- | --- | --- |
| / | 重定向 | /dashboard |
| /dashboard | 汇总、到期、趋势、入口 | stats/summary、practice-sessions |
| /words | 单词列表、编辑、导入导出 | words、words/import、words/export |

API 404 不得被 SPA 回退吞掉。session 页面 URL 中的 :id 是整数资源 ID，但页面与 API 均要求访问认证，Token 不得放入 URL。

### 8.2 页面状态

所有数据页面显式处理：

    idle → loading → success | empty | error

写操作：

    editing → submitting → submitted | conflict | error

- 409 VERSION_CONFLICT 显示服务端当前值和重新加载入口，不静默覆盖。
- 网络失败保留未保存输入，不伪装成功。
- 刷新 session 页面从服务端恢复 round 和已保存结果。
- 外部 Skill 创建的 session 与网页创建的 session 共用页面，仅显示服务端记录的 Skill 名称/版本。

### 8.3 响应式和可访问性

| 宽度 | 布局 |
| --- | --- |
| 小于 640px | 单列、底部导航、卡片列表、全宽表单 |
| 640–1023px | 可折叠侧栏，一至两列 |
| 大于等于 1024px | 固定侧栏、主内容区、多列表格 |

- 验收视口：320、375、640、768、1023、1024、1440。
- 页面主体不横向溢出；宽表格仅在自身容器 overflow-x:auto。
- 触控目标不小于 44×44 CSS 像素并适配安全区。
- 所有关键操作可用键盘完成，焦点可见；状态不能只靠颜色表达。
- 超长单词、释义和例句必须换行，不破坏屏幕或 A4 布局。
- 手机回录使用逐题卡片和固定三态按钮，不要求操作横向表格。

## 9. 安全、容量、可观测与运维约束

### 9.1 安全

- 单词、释义、例句、标签和原因始终按纯文本输出；Vue 禁止对这些字段使用 v-html。
- 默认安全头至少包含 Content-Security-Policy、X-Content-Type-Options: nosniff、Referrer-Policy 和 frame-ancestors。
- 应用自身执行请求体、导入行数和批量数量硬限制，不能只依赖反向代理。
- Token 哈希 pepper 只从只读 Secret 文件加载；数据库、镜像、Compose、前端 bundle 和日志均不得出现明文 Token。
- 认证失败、越权和写操作写审计；错误响应不泄漏资源是否存在。
- SQLite 数据目录最小权限；备份至少限制文件权限，公网或异地备份必须加密。
- readiness 不返回 schema 版本、数据库路径或异常堆栈。

### 9.2 容量与性能预算

MVP 设计容量：

- 100000 个 active word；
- 5000000 条 review_log；
- 单 session 最多 200 题；
- 单批最多 200 个结果；
- 同时活跃写请求目标不超过 5。

在上述容量和 NAS SSD 环境下的目标，而非公网 SLA：

| 操作 | p95 目标 |
| --- | ---: |
| 普通详情/分页查询 | 小于 300ms |
| 200 题生成 | 小于 2s |
| 200 项批量回录 | 小于 2s |
| readiness | 小于 200ms |

导入、导出和全量统计重建不纳入交互延迟目标。若数据量使 SQLite 无法满足目标，应先用查询计划和索引验证，再决定是否迁移数据库，不在 v1 隐式更换存储。

### 9.3 日志与保留

- 结构化请求日志：request_id、route template、status、latency、actor type；不记录 Authorization 或正文。
- audit_logs 在线保留至少 180 天；更久历史可归档。
- idempotency_records 至少保留 30 天。
- Token 和已 disabled 客户端记录至少保留到相关审计归档完成。
- 备份 RPO 目标 24 小时，更新前必须额外备份；RTO 目标 2 小时，并定期做恢复演练。

## 10. 验收矩阵

| ID | 场景 | 预期断言 |
