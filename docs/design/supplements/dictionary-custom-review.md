# 单词记忆辅助系统－词库增强与自定义复习补充设计

> 实施状态：已完成

## 1. 目标

本补充设计覆盖四个增量能力：

1. 用户只输入英文单词或上传一行一个词的 UTF-8 `.txt` 文件，系统生成音标、中文释义和英文例句；
2. 外部 `add-words` Skill 经受限 REST API 添加一个或多个单词；
3. 用户按明确单词集合生成复习表，复习表不得混入策略候选词；
4. 调试日志可通过 `request_id` 串联浏览器错误、后端请求和审计记录。

原有 CSV/JSON、单词手工编辑、策略选词和打印回录能力保持兼容。

## 2. 词典资源策略

### 2.1 当前本地资源

当前 `dictionary-index.json` 是有效 JSON 对象，共 9,189 个英文键。每项使用：

- `w`：展示词；
- `p0` / `p1`：音标候选；
- `t[]`：`pos` 与 `cn` 组成的中文释义；
- `s[]`：`c` 为英文例句，`cn` 为例句翻译。

查询使用规范化小写键，而不是依赖 `w`，因为少量词条的键和值大小写不同。补全时保留用户显式填写的字段，只填空字段；中文释义仍为空则拒绝写入，保证数据库没有半成品词条。路径由 `DICTIONARY_INDEX_PATH` 配置，默认指向仓库根目录文件。

该文件目前没有来源、作者、版本或许可证元数据，因此仅用于本地技术验证，暂不提交到 Git，也不能直接随公开镜像分发。

### 2.2 获取与更新顺序

1. **本地授权词库优先**：确认许可证后，将原始资源固定版本和校验和；构建时生成应用索引，不手工改生成文件。
2. **在线词典补缺**：仅查询本地未命中的词，并对结果做缓存、字段校验、超时和限流；供应商故障不得阻塞已有本地词条。
3. **AI 作为最后回退**：要求结构化 JSON 输出，标记来源为 AI；单词编辑器先预览，英文 TXT/文本列表导入在 `unresolved_policy=ai` 时允许校验通过后直接写入，且绝不覆盖本地或人工已有字段。
4. **人工填写兜底**：所有外部来源不可用或置信度不足时，保留英文并要求用户填写中文释义。

### 2.3 需要用户补充的内容

要把当前本地文件正式纳入仓库或镜像，需要提供：

- 原始下载地址或数据提供方；
- 词库名称、版本、发布日期和文件 SHA-256；
- 完整许可证文本，以及是否允许修改、再分发和随容器镜像发布；
- 希望采用英式音标 `p0`、美式音标 `p1`，还是按可用值回退。

若启用在线词典，还需要：

- 选定服务及官方 API 文档；
- API Key 的 Secret 保存位置；
- 每日配额、速率限制、超时、可接受成本和 NAS 是否可访问外网；
- 是否允许把用户查询词发送给第三方。

若启用 AI 回退，还需要：

- API 基础地址、模型名和 API Key Secret；生产部署通过 `AI_API_KEY_FILE` 只读挂载，兼容但不推荐明文 `AI_API_KEY`；
- 月度或单次成本上限、并发与超时；
- 数据保留要求，以及是否允许发送标签或例句上下文；
- AI 结果是“人工确认后写入”还是允许自动写入。当前单词编辑器采用人工确认，导入页面默认采用 AI 补全后自动写入。

## 3. API 与原子性

- `POST /api/v1/words/enrich`：批量预览补全，不写数据库，最多 200 词，要求 `words:write`。
- `POST /api/v1/words`：允许只给 `en_word`；补全成功后按原有幂等、唯一约束和审计规则创建。
- `POST /api/v1/words/import`：新增 TXT；空行和 `#` 注释忽略。本地未命中时按 `unresolved_policy=skip|reject|ai` 处理；`ai` 先调用 AI，仍失败的词进入 `unresolved_words`，其余词继续按原子导入事务写入。
- `POST /api/v1/daily-table/generate`：`word_ids` 非空时进入明确选择模式，校验去重、删除状态和上限，严格按给定顺序生成。

所有写请求继续使用 `Idempotency-Key`。Skill Token 至少授予 `words:write`，不得直连 SQLite。

## 4. 页面行为

- 新增单词对话框提供“自动补全”，不覆盖已填写内容；
- 导入对话框允许粘贴一行一个英文词或选择 TXT/CSV/JSON，显示词典命中数；
- 复习表生成页提供“按策略选词”和“自定义单词”，后者支持英文/中文搜索和多选；
- 自定义模式数量等于已选单词数，空选择和超过服务端上限时禁止生成。

## 5. Skill

`skills/add-words` 包含标准 `SKILL.md`、UI 元数据和纯标准库脚本。脚本只从 `WORD_MEMORY_BASE_URL` 与 `WORD_MEMORY_API_TOKEN` 读取连接信息，先调用补全预览，再逐词创建；输出创建、重复、失败和 `request_id`，不输出 Token。

## 6. 调试日志

- 前端开发模式记录请求方法、相对 URL、HTTP 状态、错误码和 `request_id`，不记录请求体或 Authorization；
- 后端记录 `request_id`、方法、路由模板、状态、耗时和 actor 类型；
- 词典记录索引加载数量、资源缺失和规范化后的未命中词；导入记录计数，不记录完整上传内容；
- 生产默认 `INFO`，诊断时临时使用 `LOG_LEVEL=DEBUG`，问题结束后恢复。

## 7. Definition of Done

- 局域网 HTTP 环境不存在 `crypto.randomUUID` 时，新增、导入和复习表生成仍可产生合法 UUID v4；
- 已授权的本地词库可补全英文单词；未知词在 AI 可用时补全并导入，AI 失败时进入可操作的未命中列表；
- TXT 文件、粘贴词表、单词对话框和 `add-words` Skill 均能完成新增；
- 自定义复习表只含明确选择的词，并保留选择顺序；
- 日志可凭页面显示的 `request_id` 定位后端请求和审计记录，且不泄露 Token；
- 前后端测试、类型检查、构建和多视口端到端测试通过。
